# Contributing to Awesome Venezuela

[Leer en Español](CONTRIBUTING.es.md)

First off, thank you for considering contributing to Awesome Venezuela! This project is a celebration of Venezuelan talent in the global tech ecosystem.

## Contribution Criteria

To keep this list high-quality and inspiring, these guidelines apply:

### 1. Venezuelan Identity
- This repo is specifically for projects built by **Venezuelans** (living in Venezuela or as part of the diaspora).
- **Mutual Trust:** To protect the privacy and security of the community, no public proof of residency or identity is required in your PR. The list relies on the honor system and the self-declaration of contributors.
- **Maintainer Discretion:** If the origin of a project is unclear or questioned, the maintainer reserves the right to request clarification through private channels or to decline the submission at their discretion to maintain the integrity of the showcase.

### 2. Originality and Creativity
- The list focuses on **truly original projects**.
- **What is NOT being sought:** Clones of generic tutorials (e.g., "PokeAPI" viewers, "To-Do" apps, generic "Weather" apps), unless they offer a significant and creative twist.
- Your project should demonstrate creativity and solve a problem or offer something interesting/new to the community.

### 3. Open Source & Public Code
- The repository must be **public** and the source code must be **visible**.
- While recognized Open Source licenses (like MIT, Apache, or GPL) are highly encouraged to ensure community reuse, any project with publicly accessible source code is eligible to be listed.

### 4. Project Health
- While there is no strict "stars" or "recent commit" count limit, the project should be functional.
- If a project becomes unmaintained for a long period (e.g., 2+ years of zero activity), it may be moved to the **Legacy & Inactive Projects** section.

## How to Submit

1. **Fork** the repository and create your branch from `main`.
2. **Add your project** to the appropriate category in `README.md`.
    - Use the existing format:
      ```markdown
      ### [Project Name](https://github.com/user/project)
      **Creator:** [Your Name](https://github.com/youruser)
      
      A clear and detailed description of the project (usually 2-4 sentences).
      ```
3. **Alphabetical Order:** Please ensure you add your project in alphabetical order within its category.
4. **Update `README.es.md`:** Since this is a bilingual project, please make sure to add the entry to the Spanish version of the README as well!
5. **Open a Pull Request:**
    - Provide a short summary of what the project does.
    - **Important:** Confirm that the project meets the **Venezuelan Identity** criteria in the PR description.

## Code of Conduct

By participating in this project, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).

Thank you for helping to showcase the best of Venezuelan engineering!
