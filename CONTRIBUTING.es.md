# Contribuir a Awesome Venezuela

[Read in English](CONTRIBUTING.md)

En primer lugar, ¡gracias por considerar contribuir a Awesome Venezuela! Este proyecto es una celebración del talento venezolano en el ecosistema tecnológico global.

## Criterios de Contribución

Para mantener esta lista con alta calidad e inspiración, se aplican estas pautas:

### 1. Identidad Venezolana
- Este repositorio es específicamente para proyectos creados por **venezolanos** (viviendo en Venezuela o como parte de la diáspora).
- **Confianza Mutua:** Para proteger la privacidad y seguridad de la comunidad, no se requiere ninguna prueba pública de residencia o identidad en tu PR. El repositorio se basa en la buena fe y la autodeclaración de las personas que contribuyen.
- **Discreción del Maintainer:** En casos donde el origen de un proyecto no sea claro o sea cuestionado, el Maintainer se reserva el derecho de solicitar aclaraciones a través de canales privados o de rechazar la solicitud a su discreción para mantener la integridad del showcase.

### 2. Originalidad y Creatividad
- La lista se enfoca en **proyectos verdaderamente originales**.
- **No se incluyen:** Clones de tutoriales genéricos (ej. visores de "PokeAPI", apps de "To-Do", apps de "Clima" genéricas), a menos que ofrezcan un giro creativo y significativo.
- El proyecto debe demostrar creatividad y resolver un problema u ofrecer algo interesante/nuevo a la comunidad.

### 3. Open Source y Código Público
- El repositorio debe ser **público** y el código fuente debe ser **visible**.
- Aunque se recomienda el uso de licencias Open Source reconocidas (como MIT, Apache o GPL) para facilitar la reutilización por parte de la comunidad, cualquier proyecto con código fuente accesible al público es elegible para la lista.

### 4. Salud del Proyecto
- Aunque no hay un límite estricto de "estrellas" o cantidad de commits recientes, el proyecto debe ser funcional.
- Si un proyecto permanece sin mantenimiento por un largo periodo (ej. 2+ años de actividad nula), podría ser movido a la sección de **Proyectos Legacy e Inactivos**.

## Cómo Enviar

1. **Haz un Fork** del repositorio y crea una rama desde `main`.
2. **Añade el proyecto** a la categoría correspondiente en `README.es.md`.
    - Usa el formato existente:
      ```markdown
      ### [Nombre del Proyecto](https://github.com/usuario/proyecto)
      **Creador:** [Tu Nombre](https://github.com/tuusuario)
      
      Una descripción clara y detallada del proyecto (usualmente de 2 a 4 oraciones).
      ```
3. **Orden Alfabético:** Por favor, asegúrate de añadir el proyecto en orden alfabético dentro de su categoría.
4. **Actualiza `README.md`:** Como este es un proyecto bilingüe, por favor asegúrate de añadir la entrada también a la versión en inglés del README.
5. **Abre un Pull Request:**
    - Proporciona un resumen corto de lo que hace el proyecto.
    - **Importante:** Confirma que el proyecto cumple con los criterios de **Identidad Venezolana** en la descripción del PR.

## Código de Conducta

Al participar en este proyecto, aceptas cumplir con el [Código de Conducta](CODE_OF_CONDUCT.es.md).

¡Gracias por ayudar a mostrar lo mejor de la ingeniería venezolana!
